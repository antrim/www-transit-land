TRANSITLAND - A SET OF MODULAR SERVICES

Onestop ID Registry is a directory of feed URLs and IDs for transit networks, stops/stations, operators/agencies, and routes. Our "Onestop" ID scheme gives global, immutable identifiers to all those different entities, so you can know when two different agencies' feeds are referring to the same bus stop. The registry is a set of JSON and GeoJSON files hosted on GitHub, ready for you to consume into your own infrastructure or to improve with a pull request.
  • Code on GitHub
Onestop Updater tends to the Onestop ID Registry. It fetches feeds, looks for updates, and generates the simple summary GeoJSON files you'll find in the registry. The updater also pushes changesets into the Transitland Community Datastore.
  • Code on GitHub
Onestop Similarity Engine is used behind the scenes, in the update loop, to determine what's changed from one set of transit data to the next. We use it as part of the Onestop Updater's automated process, but you could also use it as a simple command-line utility to "diff" two GTFS archives.
  • Code on GitHub
Transitland Community Datastore brings together data from authoritative sources (in the Onestop ID Registry) with contributions, edits, and fixes from transit enthusiasts and developers. It's a hosted service that provides a simple web API for query and editing.
  • Code on GitHub
Transitland Playground: If the Onestop ID Registry, Transitland Community Datastore, et al. are frozen and underwater, then the Playground is the tip of that iceberg, demonstrating all the pieces of the data architecture working together.
  • Code on GitHub

But we're not trying to sink ships, here. We're actively recruiting contributors, users, partners, critics, and imitators. All of Onestop's software components are available under open-source licenses; all of the data (federated from authoritative sources and contributed by community members) is freely available. You can participate in Transitland by using just one component of this architecture, or come for it all.
